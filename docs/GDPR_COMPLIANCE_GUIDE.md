# GDPR Compliance Guide — Matching Platform

## Overview

This guide covers GDPR compliance implementation. The platform collects personal data (email, location, interests, photos) and must comply with EU General Data Protection Regulation (GDPR) and CCPA.

---

## Data Export (Right of Access — Article 15 & Portability — Article 20)

### User-Triggered Export

Users can request their data from the app settings. The export is generated by `exportUserData()` and includes:

- User profile
- Extended profile
- All matches (both directions)
- All posts authored
- All conversations with sent messages
- Usage limits and subscription history

**Implementation:**
```javascript
const exportUserData = httpsCallable(functions, 'exportUserData');
const result = await exportUserData();
// Returns JSON blob — prompt user to download
```

**SLA:** Export must be delivered within 30 days of request (best practice: within 72 hours).

## Account Deletion (Right to Erasure — Article 17)

The `deleteUserData()` function implements a complete cascade:

| Data | Action |
|---|---|
| User document | Deleted |
| User's messages | Anonymized ("Message from deleted account") |
| User's matches | Deleted |
| User's posts | Archived (author_id → 'deleted_user') |
| Teams owned | Deleted |
| Team memberships | Removed from members array |
| Auth account | Deleted from Firebase Auth |
| FCM tokens | Deleted (devices subcollection) |
| Usage limits | Deleted |
| Match cache | Deleted |

**Posts are archived, not deleted** — to preserve community context and conversations that other users participated in.

**Timeline:** All data purged within 30 days (GDPR Article 17 requires "without undue delay").

## Consent Management

User consent is tracked in `consent_log` collection:

```javascript
await recordConsent(userId, 'marketing', true);   // Opt-in
await recordConsent(userId, 'analytics', false);   // Opt-out
await recordConsent(userId, 'third_party', false); // Opt-out
```

**Consent types:**
- `marketing` — Email marketing, push promotions
- `analytics` — Usage analytics (Firebase Analytics)
- `third_party` — Sharing with partners

Consent is recorded with timestamp, IP address, and consent document version.

## Data Retention Policies

| Data Category | Retention Period | Action After Expiry |
|---|---|---|
| Active user data | Until account deletion | N/A |
| Messages | 2 years from last activity | Anonymized |
| Posts | Indefinite (community) | N/A |
| Audit logs | 90 days | Deleted |
| GDPR deletion logs | 7 years (legal) | Archived to Cold Storage |
| Inactive accounts | 2 years from last login | Soft-delete prompt |

Retention is enforced by `enforceRetentionPolicy()` — call monthly via Cloud Scheduler:

```bash
# Schedule monthly retention cleanup
firebase functions:call enforceRetentionPolicy
```

## Data Minimization

The platform collects only what is necessary:
- **Required**: email (account), username (identity)
- **Optional**: location (matching), interests (matching), age (matching), photo (profile)
- **Not collected**: government ID, financial data, precise health data

## Breach Notification Procedures

Per GDPR Article 33: notify supervisory authority within **72 hours** of discovering a breach.

1. **Detect breach** → Security team notified within 1 hour
2. **Assess scope** → Within 24 hours: how many users affected, what data
3. **Notify authority** → Within 72 hours: file notification with local DPA
4. **Notify users** → Within 96 hours if high risk (e.g., passwords, location exposed)
5. **Post-incident** → Detailed report within 30 days

See `SECURITY_RUNBOOK.md` for the full incident response procedure.

---

## Privacy by Design Implementation

- **Data minimization**: Only required fields stored; optional fields nullable
- **Encryption**: Sensitive fields (phone, ID) encrypted with AES-256-GCM
- **Access control**: Firestore Security Rules enforce user-level isolation
- **Audit logging**: All sensitive operations logged in `audit_logs`
- **Anonymization**: Deleted users become "Deleted User" in conversations
- **Location precision**: Location stored as geo-point, never as address

## CCPA (California) Compliance

| Right | Implementation |
|---|---|
| Right to Know | `exportUserData()` — same as GDPR export |
| Right to Delete | `deleteUserData()` — same as GDPR deletion |
| Right to Opt-Out of Sale | `recordConsent('third_party', false)` |
| Non-discrimination | Service access not restricted by privacy choices |

**California users must see**: "Do Not Sell My Personal Information" link in app footer.
